ACE 进程限制器作用原理说明
一、程序概述
ACE 进程限制器是一款针对腾讯系进程（SGuard64.exe、SGuardSvc64.exe）的系统优化工具，参考某硕给出的教程，通过调整进程的CPU 亲和性和运行优先级，限制其对系统资源的过度占用，提升电脑运行流畅度。
二、核心优化目标
目标进程（SGuard64.exe、SGuardSvc64.exe）是腾讯安全防护相关进程，可能存在后台频繁扫描硬盘、占用大量 CPU 资源的问题。本工具降低其资源抢占能力，同时不影响其核心功能（如游戏反作弊基础功能）。
三、核心技术原理
1. 进程优先级调整
概念：操作系统通过 "优先级" 分配 CPU 时间片，优先级越高的进程获得的 CPU 资源越多。
优化逻辑：将目标进程优先级设置为系统最低级别（Windows 下为IDLE_PRIORITY_CLASS），使其仅在系统空闲时获得 CPU 资源，避免抢占其他程序（如游戏、办公软件）的资源。
效果：目标进程的 CPU 占用率会显著降低，且不会影响高优先级程序的运行流畅度。
2. CPU 亲和性限制
概念：CPU 亲和性（CPU Affinity）决定进程可以在哪些 CPU 核心上运行。现代电脑通常有多个 CPU 核心（如 4 核、8 核），默认进程可在所有核心上切换运行。
优化逻辑：将目标进程绑定到最后一个逻辑 CPU 核心（如 8 核 CPU 的第 8 个核心），限制其只能在单一核心上运行。
效果：避免目标进程在多个核心间频繁切换导致的资源浪费，同时不影响其他核心处理高优先级任务。

3. 周期性监控与调整
进程可能会被系统或自身重置优先级 / 亲和性，因此程序采用周期性检查机制：
每 180 秒扫描一次目标进程状态
若发现进程优先级或亲和性偏离目标设置，自动重新调整
首次检测到进程时会延迟 180 秒再执行限制

常见问题
为什么限制后进程仍在运行？本工具的目标是 "限制资源占用" 而非 "终止进程"，终止可能导致腾讯系程序（如游戏）无法正常运行。
如何确认限制生效？打开任务管理器（Ctrl+Shift+Esc），查看目标进程的 "优先级" 是否为 "低"，以及 "CPU" 列是否显示在指定核心（如 CPU 7）。
是否会影响游戏反作弊功能？仅调整资源分配属性，不修改进程代码或内存，不会触发反作弊机制。
